# 新しい環境の初回起動に置くサインインの道しるべ

## 背景

新しい環境を初めて起動すると、まっさらな Claude がサインインを求め、これまでのチャットがどこにも見えない。USER_GUIDE は初回サインインを繰り返し先回りして説明しているが、不安が生まれるその瞬間にアプリ内で出る案内は存在しない (grep 確認済み)。ここが初回体験で最も静かな離脱点であり、ブラウザに残っていた別アカウントでそのままサインインしてしまう取り違えの起点でもある。2026-07-03 のアイデア検討でユーザー承認済み。PR #159 の用途メモと連作で、メモに書いたアカウントをカードが引用する。

## ゴール

- 環境の初回起動のときだけ、CSW の詳細画面に 3 行の案内カードを出す。内容: まっさらでサインインを求めるのは正常であること、この環境で使うアカウントでサインインすること (メモがあれば引用)、既存のチャットと設定は既存の Claude に無傷で残っていていつでも戻れること。
- Claude Code の会話とメモリを共有またはコピーする環境では、デスクトップとローカルの区別を 1 文で補足する (デスクトップの会話はサインインしたアカウントに従う。Claude Code の会話とメモリはこの環境に引き継ぎ済み)。コピーでも会話とメモリが手元にある事実は同じため、条件は分離以外とする。
- カードは「閉じる」で消え、2 回目以降の起動では出ない。サインイン完了の検出はしない (検出にはファイルの中身を読む必要があり、原則に反するため)。

## 対象ユーザーと規模

初めて環境を作った非エンジニアを含む全ユーザー。頻度は環境ごとに 1 回だが、離脱と取り違えが実際に起きる瞬間を塞ぐ。実装はカード 1 枚と刻印 1 フィールドの軽量なもので、docs の説明だけではその瞬間に届かないため作り込みが妥当。

## 設計

- `first_launched_at` は PR #159 で導入した `state.toml` (起動状態) に持つ。`record_last_launch` が未記録のときだけ刻印する。安全宣言の `profile.toml` は起動では書き換えない原則を維持する。
- 最後の起動だけを持つ既存の `state.toml` には、最初の起動を後追いで刻印しない。この形はこの機能より前のビルドで起動済み、つまりサインイン済みの環境なので、後追いするとカードが誤表示になるため。
- カードの表示条件 (GUI): 既定の環境でない、かつ `created_at` がある (v0.20 以降に作られた環境。それ以前の環境は誤って「初回」扱いしない)、かつ `first_launched_at == last_launched_at` (起動がまだ 1 回)、かつ「閉じる」で消していない。閉じるの記録は localStorage に環境名と作成日時の組で持ち、同名で作り直した環境では改めて表示される。
- 複製した環境は `state.toml` を引き継がない (clone はプロファイル直下のファイルをコピーしない) ため、複製後の初回起動でもカードが出る。複製はサインインを引き継がないので、これは意図した挙動としてテストで固定する。
- ファイルの中身は一切読まない。判定材料は CSW 自身の刻印とユーザーが書いたメモだけ。

## リスクと対応

- リリースの同梱: 本機能は PR #159 と同じリリースに載せる。#159 だけを先にリリースすると、最初の刻印を持たない起動済み環境が生まれる。後追い刻印をしない設計によりカードの誤表示は起きないが、移行の組み合わせを増やさないため同梱を原則とする。

- 文言の事実性: デスクトップの会話はサーバー側にあり、サインインしたアカウントに従う。ローカルで引き継がれるのは Claude Code 側だけ。この区別を崩した文言を書かない (アイデア検討時の検証指摘)。
- 既存環境の誤判定: `created_at` の有無で v0.20 未満の環境を構造的に除外する。ファイルの中身は読まない。
- カードの出しすぎ: 表示は起動 1 回目の間だけ。「閉じる」の記録は UI 状態として localStorage に置き、環境のファイルには書かない。
