# 環境ごとの利用量表示の撤去と、その判断記録

## 背景

v0.17.0 / v0.17.1 で、サイドバーの各環境行に Claude のセッション上限（5時間）と週間上限の使用率を表示する機能を実装した（[usage-bars-statusline.md](usage-bars-statusline.md)）。データ源は Claude Code の statusline 連携で、通信ゼロ・認証情報非接触を守っていた。

出荷後、この機能には主対象と噛み合わない限界があると分かった。CSW の主眼はデスクトップアプリのスイートをアカウント別に分けることであり、利用者の多くは Claude Code（CLI）を常用しない。statusline 連携は Claude Code が応答したときにしか値を更新しないため、デスクトップアプリだけを使う利用者には使用率が出ない。「各環境の利用量を一覧する」という当初の狙いに、この形では届かない。

## 調査（デスクトップ分を含む使用率を、原則を守って取れるか）

CSW の絶対原則は次の 3 つである。これを 1 つも曲げずにデスクトップ利用分を含む使用率を得られるかを、既存のリソース集計ツールのソースを読んで調べた（2026-07-02）。

- インターネット通信を行わない（既定）
- 認証情報（OS の資格情報ストア・OAuth トークン・ブラウザの Cookie）に触れない
- 他アプリのローカルデータに触れない

### 実機で確認したこと

- デスクトップアプリのローカルデータ（Application Support 配下の Local Storage・IndexedDB・HTTP キャッシュ）を調べたが、使用率（5時間/週間の消費割合）を保存したファイルは存在しなかった。デスクトップアプリはサーバーから取得して画面に描くだけで、ローカルには残していない。

### 既存ツールのサーベイ結果（手法の分類）

利用量を取る手法は、認証・通信・デスクトップ分カバーの 3 軸で 4 つに分かれた。

1. ローカルの CLI ログ集計: 認証にも通信にも触れないが、取れるのはトークン数からの推定であって公式の使用率ではなく、デスクトップ分は含まれない。
2. Claude Code の statusline 連携（撤去したもの）: 公式の使用率を原則を守って取れるが、Claude Code の応答時にしか更新されず、デスクトップ単独利用では出ない。
3. 資格情報を読んで非公開の利用量 API を呼ぶ: デスクトップ分を含むが、認証情報の読み取りと通信が必須。
4. 独自にログイン/Cookie を取得して通信する: デスクトップ分を含むが、認証情報を扱い通信もする。

### 結論

デスクトップ利用分を含む使用率を取っていた手法は、例外なく認証情報の読み取りかネットワーク通信（またはその両方）を伴っていた。CSW の 3 原則を 1 つも破らずにデスクトップ分を得る手段は、現時点では存在しない。これは CSW の設計の問題ではなく、Claude 本体がデスクトップアプリ用のローカルな使用率シグナルを提供していないことに起因する。

## 判断

利便性のためにセキュリティ・プライバシーの原則を下げる実装はしない（CLAUDE.md の基本原則）。デスクトップ分が出ないまま CLI 併用者向けの機能を抱え続けるより、原則を一点の曇りもなく保つ方が CSW の価値に合う。よって利用量表示は機能ごと撤去する。利用量は Claude 本体（デスクトップアプリ・claude.ai）で確認できる。

## 撤去範囲

- `crates/core/src/usage/`（モジュールごと削除）、`lib.rs` / `error.rs` / `profile` の関連コードとテスト、`Cargo.toml` の `serde_json` を元に戻す。
- `crates/desktop` の Tauri コマンド 3 種と UI（サイドバーのゲージ、詳細画面のトグル、CSS、i18n、devInvoke モック）。
- `docs/SPECIFICATION.md` / `USER_GUIDE`（ja・en）の利用量記述。LP（ja・en）の FAQ は「現状は非対応。安全に取得する手段が無いため見送っている」旨に置き換える。
- スクリーンショットを撤去後の UI で再生成する。

## 再検討条件

次のいずれかが満たされれば、原則を守ったまま再実装できる可能性がある。そのとき本記録を起点に再検討する。

- Claude 本体が、デスクトップアプリのローカルに使用率のスナップショットを書き出す（CSW がファイルを読むだけで済む）。
- Anthropic が、認証情報の読み取りやネットワーク通信を伴わずに使用率を得られる公式の手段（ローカルコマンドやファイル出力など）を提供する。

## スコープ外

- 非公開 API の直叩き・資格情報の読み取り・Cookie の読み取りは、いずれも原則に反するため採用しない。
