# Claude Desktop Switcher — ユーザーガイド

Claude Desktop Switcherは、ClaudeデスクトップアプリとClaude Code（CLI）のアカウント環境を安全に分離・管理するためのmacOSメニューバーユーティリティです。

### なぜこのツールが必要なのか？（既存のワークアラウンドとの違い）
公式のClaudeデスクトップアプリには複数アカウントを切り替える機能がありません。そのため、多くのユーザーはターミナルから `--user-data-dir` を指定して別インスタンスを強引に起動したり、CLI専用のスイッチャー（`direnv` やエイリアス等）を使って凌いできました。
しかし、これらの方法では「デスクトップアプリの安全な分離（プロファイルごとの専用データディレクトリ）」と「CLIコンテキストの連動」を両立することはできません。

本ツールは、これらの一連のハックや複雑なシェルスクリプトを不要にし、メニューバーから**「デスクトップアプリの分離」**をワンクリックで行えます。さらに、ターミナルからコマンドを実行することで**「デスクトップアプリと連動したCLI環境の起動」**も実現。システムの環境変数を一切破壊しない「ゼロ・インパクト原則」に基づいて設計されています。

---

## 0. 考え方：「いま使っている Claude」を基準に分ける

CSW はこの一点だけ理解すれば迷いません。

- **「いま使っている Claude」= あなたの既存環境**（普段使っている Claudeデスクトップアプリと Claude Code）。これが基準点で、CSW はここを一切変更しません。
- **新しい環境を作る = 「いま使っている Claude」から、何を引き継ぎ・何を分けるかを決めること。** 共有・分離・コピーの「相手」は常に「いま使っている Claude」です。
  - **共有**: いま使っている Claude と同じ実体を使う（変更は両方に反映）
  - **分離**: この環境専用の新しい実体を持つ（独立・最初は空）
  - **コピー**: 作成時に一度だけ複製し、以後は別々
- **同じアカウントを使い続けたいだけなら、新しい環境は不要です。** その場合は「いま使っている Claude」をそのまま使えばよく、CSW を起動する必要すらありません。CSW が活きるのは「**別アカウント／別案件の独立環境**」を作りたいときです。

## 1. インストールと初期セットアップ

既存のClaude環境（デフォルトの個人環境など）はそのまま維持されます。

### ステップ 1: アプリケーションのインストール
1. GitHub の [Releases ページ](https://github.com/matsumotory/claude-desktop-switcher/releases/latest) から最新の `.dmg` ファイルをダウンロードします。
2. ダウンロードした `ClaudeDesktopSwitcher.app` を、macOSの「アプリケーション」フォルダにドラッグ＆ドロップします。
3. アプリを起動すると、画面右上のメニューバーに青いアイコンが常駐します。
4. 初回起動時のみ、設定ウインドウに 3 ステップの簡単な案内（ようこそ／使い方／ターミナル連携）が表示されます。内容を確認して閉じれば以後は表示されません。

![初回オンボーディング](../website/assets/ja_screen_onboarding.png)

### ステップ 2: 新しいプロファイル（環境）の作成とカスタマイズ
仕事用や研究用の新しい独立環境を作成します。

![Claude Desktop Switcher 設定 UI](../website/assets/ja_hero.png)

**本アプリは、データが混ざる事故を防ぐため「すべてが分離される完全隔離」がデフォルト動作となっています。**

一方で、「個人開発で育てた環境（MCP設定やルール）をそのまま使い回しつつ、トークン消費だけは仕事用アカウントに計上したい」といった高度なユースケース向けに、柔軟なカスタマイズ機能を提供しています。

1. メニューバーのアイコンをクリックし、**「設定...」**を選択します。
2. 設定ウインドウが開いたら、**「環境を作る」**ボタンをクリックします。
3. プロファイル情報を入力します。
   * **名前**: （例：`Work`, `Research` など）
   * **アイコン（任意）**: 絵文字や1文字
4. **どう分けますか?（まず2つのモードから選ぶ）**
   初見でも迷わないよう、基本は次の2択です。細かく変えたい場合だけ **「詳しく設定する（11項目を個別に）」** を開きます。

   * **まっさらに分ける（推奨・既定）**: ログインも設定も履歴もゼロから。いまの環境には一切触れません。完全に独立した別環境になります。
   * **設定だけ引き継ぐ**: いつもの MCP・ルール・スキルはそのまま使い、ログインと履歴だけ別アカウントにします。設定資産を使い回しつつ、トークンは別アカウントで消費できます。

   **＜ユースケースに合わせた選び方＞**
   * **ケースA（完全に別のプロジェクトとして作業する場合）**:
     **「まっさらに分ける」** を選びます。個人用とは切り離された、まっさらな環境が構築されます。
   * **ケースB（個人の環境を維持したまま、別アカウントへ切り替える場合）**:
     **「設定だけ引き継ぐ」** を選ぶと、慣れ親しんだ環境（MCP の設定や CLAUDE.md などのルール）をそのまま持ち込めます。

5. **「このプロファイルを作る」** を押して保存します。

![プロファイル作成ダイアログ](../website/assets/ja_screen_create.png)

> **既存プロファイルの複製**
> 設定ウインドウでプロファイルを選び **「複製」** ボタンを押すと、共有設定とプロファイル構成を引き継いだ新しいプロファイルを作成できます。似た構成をゼロから作り直す手間を省けます。ログイン状態は各コンポーネントの共有設定に依存し、隔離設定のものは引き継がれないため、新しいプロファイルで再ログインが必要な場合があります。

## 2. 日々の使い方（ワークフロー）

セットアップ完了後の、日常的な利用手順です。手作業での難しい設定などは一切不要です。

### シナリオ A: 仕事用アカウント（Work）で作業を始める
1. メニューバーのClaude Desktop Switcherアイコンをクリックします。
2. 作成した「Work」プロファイルをクリックします。
3. **新しいClaudeデスクトップのウインドウが自動的に立ち上がります。**
   （このウインドウは、完全に独立した専用のデータ領域を持っています。初回のみ、仕事用のアカウントでログインしてください）

> **ヒント: 複数のアプリを同時に起動できます**
> もともと開いていた個人用（Default）のClaudeウインドウはそのまま維持されます。個人用のウインドウで実装を進めつつ、横に並べた仕事用ウインドウでレビューを回す、といった同時進行が可能です。

### シナリオ B: ターミナル（Claude Code）を安全に起動する（エンジニア向け）
デスクトップアプリとターミナル（CLI）のアカウントを確実に一致させるための最も安全なステップです。

1. ターミナル（iTerm2や標準のTerminal）を開きます。
2. 連携コマンド `eval $(csw env <プロファイル名>)` を実行します（例: `eval $(csw env Work)`）。
3. これにより、ターミナルの環境変数が安全に「Work」環境へ切り替わります。そのまま `claude` と入力して作業を開始してください。
   （このターミナル内で消費されたトークンや履歴は、個人開発側の環境には一切干渉しません）

### シナリオ C: いつも通り（個人用）の環境に戻る
* **デスクトップ**: メニューバーから「default」プロファイルを選択するか、通常通りMacのSpotlight等から `Claude.app` を直接起動すれば、常にこれまで通りの個人用の環境が開きます。
* **CLI**: ご自身で普通に開いたターミナル（iTerm2など）で `claude` と打てば、常にデフォルト（個人用）の環境として動作します。

## 3. 知っておくべきこと（安心・安全の仕様）

* **アプリを起動し忘れても安全です（ゼロ・インパクト）**
  Claude Desktop Switcherは、システムの環境変数を裏で勝手に書き換えることはありません。本アプリを使わずに普通にClaudeを起動した場合は、100%これまで通りのデフォルト環境として動くため、既存の環境が破壊されることはありません。
* **間違ったトークン消費を防ぐ見分け方**
  「今、自分がどっちのアカウントでターミナルを叩いているか」迷った場合は、必ず `csw status` コマンドを実行してください。現在ターミナルに適用されているアクティブなプロファイル名が安全に確認できます。

## 4. よくある質問（FAQ）

**Q. 「共有」「分離」は、何に対しての共有・分離ですか？**
常に「いま使っている Claude」（あなたの既存環境）に対してです。新しい環境の各項目を、いま使っている Claude と同じものを使う（共有）か、この環境専用に新しく持つ（分離）かを選びます。

**Q. 「いま使っている Claude」とは何ですか？**
あなたが普段使っている既存の Claudeデスクトップアプリと Claude Code の環境そのものです。CSW はこれを表示・基準にするだけで、設定・履歴・ログインを変更したり削除したりしません。アプリ内ではサイドバー最上部に固定表示されます。

**Q. 同じアカウントを使うだけでも、新しい環境を作る必要がありますか？**
いいえ。同じアカウント・同じ環境を使い続けたいだけなら、新しい環境は不要です。いつもどおり Claude を起動すれば「いま使っている Claude」がそのまま使われます。CSW が役立つのは、別アカウントや別案件のために独立した環境を増やしたいときです。

**Q. 新しい環境を作ると、元の Claude は壊れませんか？**
壊れません。新しい環境はプロファイルごとの専用ディレクトリに作られ、元の環境とは物理的に分かれています。プロファイルを削除しても、元の Claude には影響しません。

**Q. 切り替えるたびに再ログインが必要ですか？**
いいえ。各環境はログイン状態を自分のディレクトリ内に保持します。初回だけログインすれば、以後は切り替えてもログインは保たれます（ただし「分離」にした項目は新しい環境では空から始まります）。

**Q. デスクトップアプリを切り替えると、ターミナル（Claude Code）も自動で揃いますか？**
いいえ。GUI と CLI のコンテキストは独立しています。ターミナルでは `eval $(csw env <プロファイル名>)` を実行して明示的に揃えてください（そのタブだけに適用され、普段の環境には影響しません）。

**Q. 「設定だけ引き継ぐ」を選ぶと、具体的に何が共有されますか？**
MCP サーバー構成・グローバルルール（CLAUDE.md）・権限/フック・プラグイン・スキル・アプリ設定・ワークツリー（設定資産）が、いま使っている Claude と共有されます。一方でログイン・会話履歴・コマンド履歴・プロジェクト記憶は安全のためこの環境専用に分離されます。細かく変えたい場合は作成画面の「詳しく設定する（11項目を個別に）」で項目ごとに調整できます。
