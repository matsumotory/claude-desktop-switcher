# ContextSwitcher for Claude — ユーザーガイド

ContextSwitcherは、ClaudeデスクトップアプリとClaude Code（CLI）のアカウント環境を安全に切り替え、トークンリソースの意図しない消費事故を防ぐためのmacOSメニューバー常駐型アプリケーションです。

既存の環境や、開発者が独自に育てたシステム設定（dotfiles等）を一切破壊しない「非破壊（ゼロ・インパクト）原則」を採用しており、オープンソースソフトウェア（OSS）として誰もが安全に利用できる設計になっています。

---

## 1. インストールと初期セットアップ

既存のClaude環境（デフォルトの個人環境）はそのまま維持されます。

### ステップ 1: アプリケーションのインストール
1. 配布された `ContextSwitcher.app` を、macOSの「アプリケーション」フォルダにドラッグ＆ドロップします。
2. アプリを起動すると、画面右上のメニューバーに「ContextSwitcher」のアイコンが常駐します。

### ステップ 2: 新しいプロファイル（環境）の作成とカスタマイズ
組織用や研究用の新しい独立環境を作成します。
**本アプリは、トークン消費事故を防ぐため「すべてが分離される完全隔離」がデフォルト動作となっています。**
一方で、「個人開発で育てた環境（MCP設定やカスタムHooks）をそのまま使い回しつつ、そこから抽出した知見の分析・検証にかかる重たいトークン消費だけは研究用（会社）アカウントに計上したい」といった高度なユースケース向けに、柔軟なカスタマイズ機能を提供しています。

1. メニューバーのアイコンをクリックし、**「Settings...」**を選択します。
2. 設定ウインドウが開いたら、**「Add Profile」**（＋ボタン）をクリックします。
3. プロファイル情報を入力します。
   * **Name**: （例：`Research`, `Company` など）
   * **Color/Icon**: メニューバーで判別しやすい色やアイコンを選択。
4. **構成管理のカスタマイズ（オプション）**
   要件に合わせて、コンポーネントごとに設定やデータの扱いを変更できます。

   * **Isolate（完全隔離）**: 【デフォルト設定】完全に独立した空の状態からスタートします。トークンもデータも互いに一切干渉しません。
   * **Share（相互同期）**: シンボリックリンクで繋ぎます。「トークンは別アカウントで消費するが、設定ファイル等は個人環境とリアルタイムに共有する」という構成を実現します。

   **＜ユースケースに合わせた設定例＞**
   * **ケースA（完全に別のプロジェクトとして作業する場合）**:
     すべてをデフォルトの **Isolate** にします。個人開発とは切り離された、まっさらな環境が構築されます。
   * **ケースB（個人開発の環境を維持したまま、研究・検証用アカウントへ切り替える場合）**:
     認証セッション自体はアプリによって強制隔離されますが、以下の設定を **Share** に変更することで、慣れ親しんだ環境をそのまま研究アカウントへ持ち込めます。
     * MCP設定 (`claude_desktop_config.json`) 👉 **Share**
     * CLI設定・Hooks (`settings.json`) 👉 **Share** （※独自スクリプトのGitトラッキングを維持できます）
     * グローバルルール (`CLAUDE.md`) 👉 **Share**

5. **「Save」**を押してプロファイルを保存します。

## 2. 日々の使い方（ワークフロー）

セットアップ完了後の、日常的な利用手順です。手作業での環境変数設定などは一切不要です。

### シナリオ A: 研究・検証用アカウント（Research）で作業を始める
1. メニューバーのContextSwitcherアイコンをクリックします。
2. 作成した「Research」プロファイルの横にある **`[ Launch Instance ]`** をクリックします。
3. **新しいClaudeデスクトップのウインドウが立ち上がります。**
   *(※このウインドウは、トークン課金が完全に独立した専用のデータ領域を持っています。初回のみ、会社用のアカウントでログインしてください)*

> **💡 ヒント（マルチインスタンス同時起動）**
> もともと開いていた個人用（Default）のClaudeウインドウは維持されます。個人開発のウインドウで実装を進めつつ、横に並べたResearchウインドウで分析を回す、といった同時進行が可能です。

### シナリオ B: ターミナル（Claude Code）を安全に起動する
DesktopとCLIのアカウントコンテキストを確実に一致させるための最も重要なステップです。

1. **必ず、目的のアカウント環境（例：Research）のClaudeデスクトップウインドウ**をアクティブ（一番手前）にします。
2. その画面内から、公式機能である**「ターミナルで開く（Open in Claude Code 等）」**の連携ボタンをクリックしてターミナルを起動します。
3. 立ち上がったターミナルでそのまま `claude` と入力して作業を開始します。
   *(※Claudeデスクトップから直接呼び出されたターミナルは、親ウインドウの隔離されたプロセス環境変数を自動的に引き継ぎます。個人開発側のトークンを意図せず消費することなく、安全に検証のCLIセッションが回せます)*

### シナリオ C: いつも通り（個人用）の環境に戻る
* **デスクトップ**: 個人用のClaudeウインドウで作業を続けるか、通常通りMacのSpotlight等から `Claude.app` を直接起動すれば、常に個人用の環境が開きます。
* **CLI**: ご自身で普通に開いたターミナル（iTerm2など）で `claude` と打てば、常にデフォルト（個人用）の環境として動作します。

## 3. 知っておくべきこと（安心・安全の仕様）

* **アプリを起動し忘れても安全です（ゼロ・インパクト）**
  ContextSwitcherは、システムの環境変数を裏で勝手に書き換えることはありません。本アプリの `[ Launch Instance ]` を使わずに普通に起動した場合は、100%これまで通りのデフォルト環境として動くため、既存のトラッキング構造が破壊されることはありません。
* **間違ったトークン消費を防ぐ見分け方**
  「今、自分がどっちのアカウントでCLIを叩いているか」迷った場合は、必ず**「そのターミナルをどのClaudeデスクトップ画面から呼び出したか」**だけを意識してください。親ウインドウのアカウント設定とコンテキストが100%確実に継承されています。
