# Claude Desktop Switcher — ユーザーガイド

Claude Desktop Switcherは、ClaudeデスクトップアプリとClaude Code（CLI）のアカウント環境を安全に切り替え、個人用や仕事用のトークンやデータが混ざってしまう事故を防ぐためのmacOSメニューバー常駐型アプリケーションです。

既存の環境やシステム設定を一切破壊しない「非破壊（ゼロ・インパクト）原則」を採用しており、オープンソースソフトウェア（OSS）として誰もが安全に利用できる設計になっています。

---

## 1. インストールと初期セットアップ

既存のClaude環境（デフォルトの個人環境など）はそのまま維持されます。

### ステップ 1: アプリケーションのインストール
1. 配布された `ClaudeDesktopSwitcher.app` を、macOSの「アプリケーション」フォルダにドラッグ＆ドロップします。
2. アプリを起動すると、画面右上のメニューバーに青いアイコンが常駐します。

### ステップ 2: 新しいプロファイル（環境）の作成とカスタマイズ
仕事用や研究用の新しい独立環境を作成します。
**本アプリは、データが混ざる事故を防ぐため「すべてが分離される完全隔離」がデフォルト動作となっています。**

一方で、「個人開発で育てた環境（MCP設定やルール）をそのまま使い回しつつ、トークン消費だけは仕事用アカウントに計上したい」といった高度なユースケース向けに、柔軟なカスタマイズ機能を提供しています。

1. メニューバーのアイコンをクリックし、**「Settings...」**を選択します。
2. 設定ウインドウが開いたら、**「+ Create Profile」**ボタンをクリックします。
3. プロファイル情報を入力します。
   * **Name**: （例：`Work`, `Research` など）
4. **構成管理のカスタマイズ（オプション）**
   要件に合わせて、コンポーネントごとに設定やデータの扱いを変更できます。

   * **Isolated（完全隔離）**: 【デフォルト設定】完全に独立した空の状態からスタートします。トークンもデータも互いに一切干渉しません。
   * **Shared（相互同期）**: 「トークンは別アカウントで消費するが、設定ファイル等は元の環境とリアルタイムに共有する」という構成を実現します。

   **＜ユースケースに合わせた設定例＞**
   * **ケースA（完全に別のプロジェクトとして作業する場合）**:
     すべてをデフォルトの **Isolated** にします。個人用とは切り離された、まっさらな環境が構築されます。
   * **ケースB（個人の環境を維持したまま、別アカウントへ切り替える場合）**:
     設定を **Shared** に変更することで、慣れ親しんだ環境（MCPの設定やCLAUDE.mdなどのルール）をそのまま持ち込めます。

5. **「Create」**を押してプロファイルを保存します。

## 2. 日々の使い方（ワークフロー）

セットアップ完了後の、日常的な利用手順です。手作業での難しい設定などは一切不要です。

### シナリオ A: 仕事用アカウント（Work）で作業を始める
1. メニューバーのClaude Desktop Switcherアイコンをクリックします。
2. 作成した「Work」プロファイルをクリックします。
3. **新しいClaudeデスクトップのウインドウが自動的に立ち上がります。**
   *(※このウインドウは、完全に独立した専用のデータ領域を持っています。初回のみ、仕事用のアカウントでログインしてください)*

> **💡 ヒント（複数のアプリを同時に起動できます）**
> もともと開いていた個人用（Default）のClaudeウインドウはそのまま維持されます。個人用のウインドウで実装を進めつつ、横に並べた仕事用ウインドウでレビューを回す、といった同時進行が可能です。

### シナリオ B: ターミナル（Claude Code）を安全に起動する（エンジニア向け）
デスクトップアプリとターミナル（CLI）のアカウントを確実に一致させるための最も安全なステップです。

1. ターミナル（iTerm2や標準のTerminal）を開きます。
2. 連携コマンド `eval $(csw env <プロファイル名>)` を実行します（例: `eval $(csw env Work)`）。
3. これにより、ターミナルの環境変数が安全に「Work」環境へ切り替わります。そのまま `claude` と入力して作業を開始してください。
   *(※このターミナル内で消費されたトークンや履歴は、個人開発側の環境には一切干渉しません)*

### シナリオ C: いつも通り（個人用）の環境に戻る
* **デスクトップ**: メニューバーから「default」プロファイルを選択するか、通常通りMacのSpotlight等から `Claude.app` を直接起動すれば、常にこれまで通りの個人用の環境が開きます。
* **CLI**: ご自身で普通に開いたターミナル（iTerm2など）で `claude` と打てば、常にデフォルト（個人用）の環境として動作します。

## 3. 知っておくべきこと（安心・安全の仕様）

* **アプリを起動し忘れても安全です（ゼロ・インパクト）**
  Claude Desktop Switcherは、システムの環境変数を裏で勝手に書き換えることはありません。本アプリを使わずに普通にClaudeを起動した場合は、100%これまで通りのデフォルト環境として動くため、既存の環境が破壊されることはありません。
* **間違ったトークン消費を防ぐ見分け方**
  「今、自分がどっちのアカウントでターミナルを叩いているか」迷った場合は、必ず**「そのターミナルをどのClaudeデスクトップ画面から呼び出したか」**だけを意識してください。親ウインドウのアカウント設定が100%確実に引き継がれています。
