# プライバシーと透明性

Claude Desktop Switcher（以下 CSW）が、あなたの Mac の何を読み、何を書き、何に決して触れないかを、この文書にすべて列挙します。あわせて、それが守られていることをあなた自身の手で確かめる方法を載せます。「安全です」と宣言するだけでは、確かめようがないからです。

この文書に書いた内容は実装と同期させています。実装を変えるときはこの文書も同じ変更で更新し、食い違いがないかをリリース前の点検で確認しています。

## 4 つの約束

1. **インターネット通信を行いません。** 利用状況の送信も、自動アップデートの通信もありません。
2. **パスワードやサインイン情報を読み取りません。** パスワードを保管する macOS のキーチェーンに触れず、サインインのトークンやブラウザの Cookie の中身を開くこともありません。
3. **他のアプリのデータに触れません。** 既存の Claude のデータを勝手に読み書きしません。触れるのは、あなたが環境の作成時に共有やコピーを選んだ項目だけです。
4. **既定では何も書き換えません。** シェルの設定ファイルも、OS の環境変数も、既存の Claude も、そのままです。CSW を使わずに Claude を起動すれば、これまでどおりの Claude が動きます。

## 通信について

CSW はインターネットに接続しません。接続する仕組み自体を持っていません。

唯一の例外は、画面下部のリンク（ユーザーガイド・問題を報告・更新を確認）と、「このアプリについて」の画面にある「確かめ方を見る」です。これらを押すと、あらかじめ決めてある GitHub の URL を macOS に渡し、既定のブラウザで開きます。通信するのはブラウザであって、CSW 自身ではありません。渡せる URL は次の 5 つに固定してあり、実装のうえでも `https` 以外の URL を拒否します。

- `https://github.com/matsumotory/claude-desktop-switcher/blob/main/docs/USER_GUIDE.md`
- `https://github.com/matsumotory/claude-desktop-switcher/issues`
- `https://github.com/matsumotory/claude-desktop-switcher/releases`
- `https://github.com/matsumotory/claude-desktop-switcher/blob/main/docs/PRIVACY.md`
- `https://github.com/matsumotory/claude-desktop-switcher/blob/main/docs/PRIVACY_EN.md`

更新の確認も同じ仕組みです。CSW が新しいバージョンを問い合わせるのではなく、GitHub のリリースページをブラウザで開くだけです。

## CSW が書く場所

CSW がファイルを書き込むのは、自分専用のフォルダ `~/.context-switcher-claude/` の中だけです。

| パス | 内容 |
|---|---|
| `~/.context-switcher-claude/config.toml` | いま選ばれている環境の名前 |
| `~/.context-switcher-claude/profiles/<環境名>/profile.toml` | その環境の設定（名前・共有モード） |
| `~/.context-switcher-claude/profiles/<環境名>/desktop-data/` | その環境の Claude Desktop App 用データ |
| `~/.context-switcher-claude/profiles/<環境名>/cli-data/` | その環境の Claude Code 用データ |

このほかに、設定ウィンドウの表示設定（選んだアクセントカラーと、初回案内を表示したかどうかの記録）を、macOS がアプリごとに用意する画面データの保存場所に保存します。ここに入るのは CSW の見た目の設定だけで、あなたのデータや Claude のデータは含まれません。

## CSW が読む場所

- 上の表にある自分専用のフォルダは、読み書きの両方を行います。
- 既存の Claude のフォルダ `~/Library/Application Support/Claude` と `~/.claude` は、次の 3 つの場合にだけ触れます。
  1. **存在の確認**: フォルダやファイルがあるかどうかを調べます。中身は読みません。
  2. **共有リンクの作成**: 環境の作成時に共有を選んだ項目について、新しい環境の側に、既存の Claude の実体を指すシンボリックリンクを作ります。リンクを作るのは新しい環境のフォルダの中で、既存の Claude 側には何も書きません。
  3. **コピー**: 環境の作成時にコピーを選んだ項目について、その項目のファイルを読み取り、新しい環境へコピーします。読み取るのはあなたが選んだ項目だけで、既存の Claude 側は変更しません。
- 起動中の Claude を見分けるために、実行中のプロセスの一覧と起動引数を読みます。どのデータフォルダで Claude が動いているかを知るためで、プロセスの中身や通信内容は読みません。

## 環境の複製について

環境の詳細画面にある複製は、選んだ環境のフォルダ全体を、あなたの Mac の中で新しい環境のフォルダへそのまま写す機能です。写す中にはその環境のサインイン状態も含まれるため、複製した環境は同じアカウントでサインインした状態で始まります。このときも CSW が行うのはファイルをそのまま写すことだけで、中身を解釈したり、外部へ送ったりすることはありません。既存の Claude を複製することはできません。

## CSW が実行する OS コマンド

CSW は次の 4 つの macOS 標準コマンドだけを実行します。いずれもローカルの操作で、通信しません。

| コマンド | 何のために使うか |
|---|---|
| `open` | 選んだ環境のデータフォルダを指定して Claude を起動します。リンクを既定のブラウザで開くのにも使います |
| `pgrep` | Claude Desktop App が起動中かどうかを調べます |
| `ps` | 起動中の Claude がどの環境のデータフォルダを使っているかを、起動引数から読み取ります |
| `hdiutil` | CSW のインストール用ディスクイメージがマウントされたままのとき、それを検出して取り出します |

## 決して触れないもの

- パスワードを保管する macOS のキーチェーン。読み書きするコード自体がありません。
- サインインのトークン、ブラウザや Claude の Cookie。各環境のサインイン情報はその環境のフォルダの中にあり、CSW がその中身を開いて解釈することはありません。環境の複製で写すときも、そのまま写すだけです。
- 上の「CSW が読む場所」に挙げた以外の、他のアプリのデータ。
- ネットワーク。送信も受信もしません。

## 自分で確かめる方法

### 通信していないことを確かめる

CSW を起動し、設定ウィンドウを開いた状態で、ターミナルで次を実行します。

```bash
lsof -i -a -p $(pgrep -f Claude-Desktop-Switcher | head -1)
```

このコマンドは、CSW のプロセスが開いているインターネット接続を一覧表示します。何も表示されなければ、接続はありません。

正確を期すための補足です。設定ウィンドウの描画は、macOS の仕組みにより WebKit という別プロセスが担います。CSW の画面はアプリに同梱したローカルファイルだけを読み込むため、そこにも外部への接続はありません。ただし、上のコマンドはアプリ本体のプロセスだけを見るため、この Mac 全体の通信まで含めて徹底的に確認したい場合は、`nettop` や通信を監視する専用アプリで、CSW に関係する接続が現れないことを観察してください。

### 配布物が本物であることを確かめる

Apple にアプリの身元を確認してもらう審査を公証といいます。ダウンロードした CSW がこの審査を通ったものであることは、次のコマンドで確認できます。

```bash
codesign -dv --verbose=2 "/Applications/Claude-Desktop-Switcher.app"
spctl --assess --type execute -v "/Applications/Claude-Desktop-Switcher.app"
```

`spctl` が `accepted` と `Notarized Developer ID` を表示すれば、Apple の公証を通った配布物です。

### 分離とリンクの実体を確かめる

環境のデータが実際にどうつながっているかは、Finder やターミナルでそのまま見られます。

```bash
ls -la ~/.context-switcher-claude/profiles/<環境名>/cli-data/
```

共有を選んだ項目は、既存の Claude の実体を指すシンボリックリンクとして表示されます。分離した項目は、その環境専用の実体ファイルとして表示されます。

### 依存ライブラリに通信機能が無いことを確かめる

「通信しない」を約束として書くだけでなく、依存ライブラリの段階で機械的に強制しています。リポジトリの `deny.toml` に、HTTP や WebSocket のクライアントライブラリの一覧を禁止として登録してあり、これらが依存に入ると CI のビルドが失敗します。この検査はすべてのプルリクエストで自動実行されます。

正直に書いておくべき点がひとつあります。依存関係の記録ファイルである `Cargo.lock` には、`reqwest` と `hyper` という HTTP ライブラリの項目が載っています。これは、CSW の画面基盤である Tauri が Windows と Linux 向けの依存として宣言しているためで、macOS 向けのビルドには含まれません。この事実はソースコードから次のコマンドで確認できます。

```bash
cargo tree --target aarch64-apple-darwin -i reqwest
cargo tree --target aarch64-apple-darwin -i hyper
```

`warning: nothing to print.` と表示されれば、そのライブラリは macOS 向けの依存関係に含まれません。

### 配布物の部品表（SBOM）を確かめる

v0.19.0 以降の各リリースには、配布物に含まれる全ライブラリの一覧を CycloneDX 形式で添付します。GitHub のリリースページから `csw-desktop_aarch64-apple-darwin.cdx.json` などのファイルをダウンロードすると、macOS ビルドに含まれるすべての部品とバージョンを確認できます。
