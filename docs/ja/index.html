複数のClaude環境を、安全かつ完全に分離。

パワーユーザーのためのmacOSメニューバーユーティリティ。個人、業務、研究のコンテキストを完全に切り離し、APIトークンの流出や履歴の混入を根本から防ぎます。

macOS版をダウンロード ドキュメントを読む

完全なデータアイソレーション

APIトークン、チャット履歴、コンテキストなどすべての情報を環境ごとに厳密に隔離します。業務環境で個人のトークンを誤って消費してしまうような事故はもう起きません。

既存環境へのゼロ・インパクト

システムの環境変数を汚染したり、既存の設定を上書きすることはありません。すべてはサンドボックス化されたパスで安全に実行されます。

スマートな同期設定

会話履歴は隔離しつつも、CLAUDE.mdのグローバルルールやインストール済みのプラグインなど、共通化したい設定だけをプロファイル間で同期することが可能です。

無駄のないワークフロー

複雑な初期設定は不要です。すべてメニューバーから直感的に操作できます。

ユーザーガイド

より詳細な使い方や、隔離・同期の仕組みについて

1. インストールと初期セットアップ

既存のClaude環境（デフォルトの個人環境など）はそのまま維持されます。

  1. 配布された ClaudeDesktopSwitcher.app を、macOSの「アプリケーション」フォルダにドラッグ＆ドロップします。
  2. アプリを起動すると、画面右上のメニューバーに青いアイコンが常駐します。

新しいプロファイル（環境）の作成とカスタマイズ

仕事用や研究用の新しい独立環境を作成します。本アプリは、データが混ざる事故を防ぐため「すべてが分離される完全隔離」がデフォルト動作となっています。

  1. メニューバーのアイコンをクリックし、「Settings...」を選択します。
  2. 設定ウインドウが開いたら、「+ Create Profile」ボタンをクリックします。
  3. Name にプロファイル名（例：Work, Research）を入力します。
  4. 構成をカスタマイズ（オプション）します。
    • Isolated（完全隔離）: デフォルト設定。完全に独立した空の状態からスタートします。
    • Shared（相互同期）: トークンは別アカウントで消費しつつ、設定ファイル等（MCP設定等）は元の環境と同期します。
  5. 「Create」を押してプロファイルを保存します。

2. 日々の使い方（ワークフロー）

シナリオ A: 仕事用アカウントで作業を始める

  1. メニューバーから作成した「Work」プロファイルをクリックします。
  2. 新しいClaudeデスクトップのウインドウが自動的に立ち上がります。※初回のみログインが必要です。

シナリオ B: ターミナル（Claude Code）を安全に起動する

  1. 必ず、目的のアカウント環境（例：Work）のClaudeデスクトップウインドウを一番手前にします。
  2. その画面内から「ターミナルで開く」連携ボタンをクリックしてターミナルを起動します。
  3. そのまま claude と入力して作業を開始します。（自動的に仕事用環境として認識されます）

シナリオ C: いつも通りの個人用環境に戻る

Spotlight等から通常通り Claude.app を起動するか、普通のターミナルで claude と打てば、常にデフォルト（個人用）の環境として動作します。

3. 安心・安全の仕様（ゼロ・インパクト）

Claude Desktop Switcherは、システムの環境変数を裏で勝手に書き換えることはありません。本アプリを使わずに普通にClaudeを起動した場合は、100%これまで通りのデフォルト環境として動くため、既存の環境が破壊されることはありません。
