# EU AI Act Compliance-by-Design

## Abstract
Enforceable as of August 2026, Article 50 of the EU AI Act mandates strict transparency and record-keeping for AI systems. This skill enforces "Compliance-by-Design" for AI agents generating code or interacting with systems.

## Core Directives

### 1. Audit Trails (Record Keeping)
* **Immutable Logs:** Every significant architectural decision, code generation step, and command execution performed by the AI agent MUST be logged in a persistent artifact (e.g., `walkthrough.md` or a dedicated audit file).
* **Decision Provenance:** When writing complex algorithms or security-sensitive code, the agent must document *why* a specific approach was chosen, linking to the original human requirement or standard constraint.

### 2. Transparency Obligations (Article 50)
* **AI Authorship Disclosure:** Any non-trivial documentation, user-facing copy, or generated assets (like READMEs or User Guides) must include a disclaimer that it was generated or assisted by an AI system.
* **Watermarking/Tagging:** In source code, large blocks of purely AI-generated boilerplate should be tagged with standard comments (e.g., `// Generated by AI Agent`) if requested by the repository policies.

### 3. Safety and Security Guardrails
* **No Covert Execution:** The agent must never attempt to obfuscate its actions or hide execution steps from the human orchestrator.
* **Explicit Escalation for High-Risk Tasks:** If a task touches cryptographic keys, authentication databases, or live production environments, the agent MUST halt execution, explicitly state the risk, and demand human approval.
